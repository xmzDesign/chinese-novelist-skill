# 追读理由与吸引力门禁

本指南定义 Novel Harness 的吸引力标准。目标是让小说不仅合格，而且有亮点、幽默感和持续阅读动力。

## 核心规则

章节通过必须同时满足：

```text
readerHookStatus == "pass"
readerHookScore >= 80
memorableMoment 非空
chapterTurnPageHook 非空
endingStrategy 合法
formulaicIssues 为空
highlightIssues 为空
```

第 1-3 章更严格：

```text
readerHookScore >= 85
goldenThreeStatus == "pass"
```

幽默不是每章强行讲笑话。严肃题材也要有“可读性调味”：人物反差、嘴损、误会、尴尬、荒诞感、冷反应或局部轻松。

## 章节必须有亮点

每章至少要有 1 个能被读者记住的点：

| 类型 | 说明 |
|---|---|
| 爽点 | 打脸、反击、证明自己、拿到关键线索、关系推进 |
| 笑点 | 角色嘴损、处境反差、误会、尴尬、冷幽默 |
| 反转 | 信息反转、行动反转、身份反转、立场反转 |
| 名场面 | 有画面、有台词、有情绪记忆点的桥段 |
| 人物魅力 | 主角聪明、可爱、锋利、嘴欠、执着、反差 |
| 情绪释放 | 憋屈后的局部释放，或压抑后的短暂喘息 |

如果一章没有可复述的亮点，即使结构正确，也不能通过追读理由门禁。

## 幽默规则

幽默必须来自人物和处境，不能靠硬塞段子。

可用方式：

- **嘴损但不油**：角色用短句、偏见、误判或犀利反应制造笑点。
- **处境反差**：严肃目标遇到荒唐障碍。
- **误会推进**：误会必须推动剧情或关系，不只是插科打诨。
- **尴尬真实**：人物嘴硬、社死、装镇定失败。
- **冷反应**：别人崩溃时，某人用异常冷静的反应制造反差。

禁用方式：

- 为了搞笑让角色突然降智。
- 长段解释笑点。
- 网络烂梗堆砌。
- 和题材基调冲突的无关段子。

## 追读力评分

| 维度 | 分值 | 检查重点 |
|---|---:|---|
| 开场抓力 | 15 | 前 300 字是否有行动、异常、冲突或强钩子 |
| 章节亮点 | 20 | 是否有至少 1 个可复述的爽点、笑点、反转或名场面 |
| 幽默与反差 | 10 | 是否有符合基调的人物反差或轻微幽默 |
| 主角魅力 | 15 | 主角是否显出聪明、执着、嘴损、善良、强行动力等吸引点 |
| 爽点兑现 | 15 | 是否有局部兑现，而不是一直铺垫 |
| 期待升级 | 15 | 结尾是否让问题更大、奖励更近或危险更具体 |
| 阅读顺滑 | 10 | 是否少废话、少停滞、场景推进清楚 |

## 结尾策略与反套路

`chapterTurnPageHook` 的含义是“读者继续读的理由”，不是“每章最后必须抛一个硬悬念”。允许的 `endingStrategy`：

| 策略 | 说明 |
|---|---|
| `payoff-close` | 本章完成一次兑现，用余味和下一步奖励驱动追读 |
| `soft-question` | 留一个轻问题，不制造廉价危机 |
| `decision-point` | 主角做出选择，读者想看代价 |
| `emotional-aftertaste` | 用情绪余波、关系裂痕或温度承接 |
| `resource-reveal` | 揭示新资源、新线索、新规则 |
| `relationship-shift` | 人物关系发生可感知变化 |
| `threat-approach` | 危险逼近，但必须具体且有铺垫 |

禁用的机械化结尾：

- 连续章节都用黑影、电话、门响、短信、冷笑、血字等同质钩子。
- 每章最后一句都写“他还不知道，更大的危险正在靠近”。
- 本章没有兑现，只靠“下一章会发生事”强行追读。
- 连续两章以上使用同一种强悬念策略。

机械化结尾问题使用 `M-XX` 编号：

| 编号 | 类型 |
|---|---|
| M-01 | 连续强悬念结尾 |
| M-02 | 惯用物件钩子重复 |
| M-03 | 只铺垫不兑现 |
| M-04 | 结尾和本章核心事件无因果 |
| M-05 | 追读理由只靠“下一章会发生事” |

## 爽文专项

所有章节都执行爽文专项。爽文不是固定堆 3-5 个爽点，而是检查本章是否完成有效情绪兑现。

必须满足：

```text
shuangwenStatus == "pass"
satisfactionBeats 非空
shuangwenIssues 为空
```

有效 `satisfactionBeats` 可以是：打脸、反击、升级、拿到资源、获得线索、关系推进、误判反转、憋屈释放。每个爽点都必须有铺垫、行动、兑现和后续期待。

爽文专项问题使用 `S-XX` 编号：

| 编号 | 类型 |
|---|---|
| S-01 | 爽点无铺垫 |
| S-02 | 本章无有效爽点节拍 |
| S-03 | 只憋屈不释放 |
| S-04 | 主角升级不可见 |
| S-05 | 打脸没有因果 |
| S-06 | 爽点靠旁白宣布 |

## Reader Hook 问题编号

QA 报告中的追读问题用 `R-XX` 编号：

| 编号 | 类型 |
|---|---|
| R-01 | 开场抓力不足 |
| R-02 | 本章没有可记忆亮点 |
| R-03 | 幽默缺席或幽默破坏基调 |
| R-04 | 主角魅力没有展示出来 |
| R-05 | 爽点只铺垫没有兑现 |
| R-06 | 结尾追读理由不成立 |
| R-07 | 章节中段停滞 |
| R-08 | 反转或冲突太常规 |

## Evaluator 输出要求

每个失败项必须写清：

- 位置或证据
- 为什么不吸引人
- 会导致读者在哪一步流失
- 具体修复建议

示例：

```markdown
### R-02：本章没有可记忆亮点

- **位置或证据**：全章都在解释调查流程，没有一个让读者能复述的桥段
- **为什么不吸引人**：信息推进有了，但缺少爽点、笑点或反转
- **流失风险**：读者读完只知道“查了线索”，没有期待下一章的情绪
- **修复建议**：增加一个主角用非常规方法套出线索的桥段，让配角误判主角，最后被一句反问打脸
```

## Fix Writer 修复规则

| 问题 | 修法 |
|---|---|
| 开场弱 | 前 300 字加入行动、异常、冲突或强信息差 |
| 没亮点 | 增加可复述桥段：反击、误会、反转、名台词 |
| 幽默弱 | 从人物性格里生笑点，不插无关段子 |
| 主角不吸引 | 让主角做一个有锋芒、有反差或有代价的选择 |
| 爽点不足 | 给局部兑现，保留更大问题 |
| 结尾弱 | 强化追读理由；可以让危险更具体、奖励更近、秘密更反常，也可以用兑现余味或关系变化承接 |
| 机械化结尾 | 更换 `endingStrategy`，把强悬念改成兑现收束、情绪余味或关系变化 |
| 爽文弱 | 补一个有铺垫的局部兑现，明确主角获得了什么和下一步想要什么 |

修复不能破坏章节契约、人物一致性、黄金三章和反 AI 门禁。
