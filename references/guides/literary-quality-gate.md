# 文学质量与反 AI 门禁

本指南定义 Novel Harness 的最低文学质量标准。目标不是追求纯文学奖项，而是守住底线：读者不应一眼看出这是 AI 写的。

## 核心规则

章节通过必须同时满足：

```text
antiAiStatus == "pass"
literaryScore >= 80
blockingIssues 为空
```

第 1-3 章更严格：

```text
antiAiStatus == "pass"
literaryScore >= 85
goldenThreeStatus == "pass"
blockingIssues 为空
```

不要依赖外部 AI 检测器分数。Evaluator 必须用文本证据说明哪里像 AI、为什么像、怎么改。

## 反 AI 一票否决项

以下任一项严重出现时，`antiAiStatus` 必须为 `fail`：

| 问题 | 判定方式 | 修复方向 |
|---|---|---|
| 人物说话同质化 | 不同角色台词像同一个人在说话 | 给角色区分句长、词汇、回避方式和口头习惯 |
| 情绪全靠概括 | 大量“复杂情绪”“难以言喻”“前所未有” | 改成动作、停顿、物件、身体反应或失控台词 |
| 空泛形容堆砌 | “璀璨、瑰丽、深邃、命运、复杂”等密集出现 | 删除，不用同义词替换 |
| 四字套话过多 | 心潮澎湃、命运齿轮、风起云涌、前所未有 | 改成更具体、更口语或更锋利的表达 |
| 段落节奏模板化 | 每段长度接近，结构总是动作+心理+环境 | 打散段落长度，增加短促反应和留白 |
| 解释替代表演 | 直接解释人物性格、冲突、主题 | 让事件、选择、对话和后果呈现 |
| 低级连续性错误 | 人名、时间、视角、设定、动机冲突 | 回到大纲、人物档案和连续性文件修正 |
| 爽点或冲突缺席 | 3000 字以上但没有有效事件推进 | 增加阻力、选择、代价、误判或局部兑现 |

## AI 痕迹问题编号

QA 报告中的反 AI 问题用 `A-XX` 编号：

| 编号 | 类型 |
|---|---|
| A-01 | 抽象情绪替代表演 |
| A-02 | 角色台词同质化 |
| A-03 | 空泛形容和四字套话堆积 |
| A-04 | 段落节奏过于工整 |
| A-05 | 设定解释压过剧情 |
| A-06 | 作者替人物总结主题 |
| A-07 | 低级连续性错误 |
| A-08 | 冲突或爽点缺席 |

## 文学质量评分

| 维度 | 分值 | 检查重点 |
|---|---:|---|
| 画面具体性 | 15 | 是否有具体动作、物件、场景细节，而不是抽象描述 |
| 人物声音 | 15 | 台词是否有角色差异、潜台词和关系张力 |
| 情绪可信度 | 15 | 情绪是否由事件、行为和代价自然推出 |
| 冲突密度 | 15 | 是否持续存在阻力、选择、代价和变化 |
| 节奏变化 | 10 | 长短句、场景推进、停顿和留白是否自然 |
| 语言自然度 | 15 | 是否少套话、少 AI 腔，表达具体而不端着 |
| 类型爽点 | 10 | 是否兑现本题材读者期待 |
| 余味与钩子 | 5 | 结尾是否让人想翻下一章 |

## Evaluator 输出要求

Evaluator 不能只写“文笔像 AI”。必须写：

```markdown
### A-01：抽象情绪替代表演

- **位置或证据**：正文第 6 段，“复杂情绪”“难以言喻”连续出现
- **为什么像 AI**：用概括词跳过了人物真实反应
- **修复建议**：改成主角的具体动作、停顿、手部反应或一句失控台词
```

每个失败项必须满足：

- 有位置或证据
- 有问题类型
- 有“为什么像 AI”
- 有具体修复建议

## Fix Writer 修复规则

| 问题 | 修法 |
|---|---|
| 空泛心理 | 改成动作、物件、身体反应、短句台词 |
| AI 套话 | 直接删除或换成具体事实，不做同义词替换 |
| 对话同质 | 为角色区分句长、词汇、回避方式和攻击方式 |
| 设定解释 | 拆进冲突、误会、道具、任务失败或人物争执 |
| 节奏太平 | 增加选择、代价、误判、时间压力 |
| 爽点不足 | 让主角赢一点，但付出代价或留下更大坑 |

修复必须保留已通过的核心事件、人物关系、伏笔和结尾策略。若失败项是机械化结尾，可以调整追读理由，但不能破坏已通过的因果承接。
