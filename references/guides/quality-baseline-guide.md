# 质量基准指南

质量基准用于在写正文前确定“这本书到底要给读者什么”。它不是文案包装，而是后续大纲、章节契约、场景卡、QA 和 Editor Gate 的共同判断标准。

## 目的

很多低质量章节不是文笔问题，而是写之前没有统一的判断尺：

- 目标读者不清楚，导致节奏和表达摇摆。
- 作品承诺不清楚，导致每章都在写事件，但读者不知道为什么要追。
- 禁忌写法不清楚，导致模型反复使用同质套路。
- 参考作品只学表面桥段，没有学吸引力机制。

质量基准要在 Phase 2 生成，并在 Phase 3 每章写作前读取。

## `05-质量基准.md` 必填内容

```markdown
# 质量基准

## 目标读者
- **核心读者**：[谁会读这本书]
- **阅读场景**：[碎片时间/沉浸阅读/睡前/通勤等]
- **主要需求**：[爽感/悬疑/情绪陪伴/成长/恋爱张力/世界探索]
- **耐心阈值**：[读者能接受几章铺垫，什么时候必须兑现]

## 作品核心承诺
- **一句话承诺**：[读者打开这本书，期待持续获得什么]
- **前三章留存理由**：[第1章看什么，第2章为什么不走，第3章为什么追下去]
- **长期追读理由**：[10章以后还想看的原因]

## 质量红线
- **绝对不写**：[低智反派/流水账/空泛抒情/硬解释设定/机械钩子等]
- **弱化处理**：[题材中容易过量但需要克制的元素]
- **必须保留**：[本书最核心的爽点、情绪或类型味道]

## 参考作品学习清单
| 参考 | 学什么 | 不学什么 |
|---|---|---|
| [作品/作者] | [节奏/人物/爽点/悬念/语言] | [不适合本书的部分] |

## 质量评分优先级
1. [例如：追读感]
2. [例如：主角魅力]
3. [例如：爽点兑现]
4. [例如：文字自然度]

## 读者流失雷区
| 流失点 | 触发条件 | 预防方式 |
|---|---|---|
| 开头慢 | 前300字没有行动、异常、冲突或强信息差 | 从动作/异常/选择切入 |
| 中段空转 | 连续1000字没有新信息、新选择或新阻力 | 每个场景都让局面变化 |
| 爽点廉价 | 对手降智或主角靠旁白赢 | 补铺垫、阻力和反应 |
```

## 使用方式

Story Planner 生成大纲和章节契约时，必须对照质量基准：

- 每章至少服务一个核心承诺。
- 前三章必须验证“留存理由”。
- 章节契约必须写清该章避免哪些流失雷区。
- 场景卡必须能说明每个场景如何让局面发生变化。

Evaluator 和 Editor Gate 不能只看“有没有完成剧情”，还要判断本章是否兑现质量基准中的读者承诺。

## 常见失败

| 失败 | 表现 | 修复 |
|---|---|---|
| 目标读者过泛 | 同时想写爽文、文学、悬疑、治愈，节奏混乱 | 缩到一个核心读者和一个主需求 |
| 承诺不可执行 | “写一个好故事”“让读者喜欢” | 改成可检验的持续体验 |
| 禁忌写法太空 | “不要写差” | 写成具体禁用行为 |
| 参考作品误用 | 照抄桥段或语气 | 只提取机制，不复制内容 |

质量基准不通过时，不进入章节正文写作；先修基准、大纲或章节契约。
