# 网文顶层设计指南

本指南定义 Novel Harness 的网文吸引力底层逻辑。它不是额外题材，也不是“无脑爽文”开关，而是所有长篇创作默认要满足的设计层：让虚构设定可代入，让欲望兑现可感知，让读者一直知道下一步为什么值得看。

## 三个读者引擎

### 1. 代入感：合理化不合理

读者可以接受离奇、反常识、夸张的设定，但必须看见它为什么在本书里成立。写作时必须给“不合理”补上至少一种合理化支点：

| 支点 | 写法 |
|---|---|
| 现实共鸣 | 对应读者熟悉的委屈、欲望、压力、职业处境或关系难题 |
| 规则解释 | 设定有触发条件、边界、代价或可验证结果 |
| 证据链 | 通过物件、记录、旁人反应、失败案例证明设定不是作者随口说 |
| 情绪出口 | 主角替读者完成现实中不敢做、不能做、做不到的反击或选择 |
| 代价约束 | 任何强能力、好运气或特殊身份都要有风险、限制或后续麻烦 |

不要用大段设定说明来“解释合理”。合理化应该进入场景：有人质疑、有人误判、主角付出代价、规则造成后果，读者自然接受。

### 2. 爽点：核心满足感

网文爽点的核心是欲望升级：财富、权力、能力、身份、关系、尊严、自由或真相的获得。有效爽点必须有过程，不是旁白宣布主角厉害。

推荐节拍：

```text
压迫/误判 -> 主角用熟悉能力或规则行动 -> 局部兑现 -> 旁人反应 -> 更大目标或更难阻力
```

爽点可以小，但必须可见。每章至少给一次有效情绪兑现；大爽点可以跨 2-3 章蓄势，但中途要有局部回收，不能连续憋屈。

### 3. 期待感：挖坑与填坑

每章都要同时处理两件事：

- **填坑**：兑现上一章或本弧段承诺给读者的情绪、信息、爽点或关系推进。
- **挖坑**：制造下一章的具体期待，让读者想知道奖励、危险、秘密、选择或关系变化会怎样发展。

追读理由不等于每章最后一句硬悬念。已兑现后的余味、奖励临近、规则揭示、关系转向、主角选择和更高层对手出现，都可以成为期待。

## 故事结构与创作法

### Core Loop：核心循环

长篇必须有可重复、可升级的主角行动循环。不同题材循环不同，但结构相同：

```text
进入问题/区域 -> 解决冲突 -> 获得收益 -> 主角升级 -> 进入更难问题/区域
```

示例：

| 题材 | 核心循环 |
|---|---|
| 玄幻/游戏 | 去怪物区 -> 击败强敌 -> 获得经验/装备 -> 升级 -> 进入更危险区域 |
| 悬疑 | 接触异常 -> 找到线索 -> 识破误导 -> 获得权限/真相碎片 -> 进入更深案件 |
| 都市经营 | 发现需求 -> 调动资源 -> 完成交易/产品 -> 获得资金和人脉 -> 进入更大市场 |
| 职场逆袭 | 遭遇不公 -> 用专业能力解决问题 -> 获得认可/筹码 -> 升级职位或话语权 -> 面对更高层博弈 |

每章契约必须标明本章处在循环哪一步，避免长篇中段变成随机事件堆叠。

### 黄金开局与前三章

前三章必须快速交代三件事：

1. **核心梗**：本书最吸引人的设定、身份、矛盾或反差。
2. **金手指/优势来源**：可以是系统、重生经验、职业能力、信息差、资源、人脉或性格锋芒。
3. **初始冲突**：让主角立刻进入压力，不要长时间日常铺垫。

第 1 章让读者知道“这本书看什么”；第 2 章让冲突不可逆；第 3 章给第一次有限兑现，并打开更大的循环。

### 合理化主角

主角最好先做自己熟悉或能合理迁移的事。新能力可以强，但必须和主角原有经验、性格、职业、记忆或训练建立桥梁。

常见失败：

- 年纪很小却无铺垫地拥有复杂权谋和行业经验。
- 主角突然精通陌生领域，只因为作者需要他赢。
- 反派集体降智，衬托主角厉害。
- 主角选择没有代价，所有人无条件配合。

修复方式：

- 给主角一个熟悉领域或能力来源。
- 让胜利依赖具体行动、观察、资源调度或规则利用。
- 让对手保留合理阻力，主角赢得有价值。

### 系统流逻辑

系统、金手指、重生、模拟器、面板、签到、任务奖励等设定可以减少前期铺垫，但不能没有内在规则。至少明确：

| 规则 | 问题 |
|---|---|
| 触发条件 | 什么时候会出现任务、奖励或能力变化？ |
| 奖励边界 | 能给什么，不能给什么？ |
| 成长路径 | 主角如何从小收益升级到大收益？ |
| 代价限制 | 使用能力会消耗什么、暴露什么或引来什么麻烦？ |
| 情节参与 | 系统规则如何推动选择、冲突和爽点，而不是只发通知？ |

非系统文也要有类似规则：职业流程、资源交换、权力秩序、魔法体系、案件逻辑、商业规则或社会规则。

## 规划产物要求

Phase 2 必须生成 `04-网文顶层设计.md`，至少包含：

- 读者现实共鸣与情绪出口
- 不合理设定及合理化规则
- 主角熟悉领域与优势来源
- 核心循环名称、步骤和升级收益
- 系统/金手指/能力规则
- 爽点阶梯与阶段性大兑现
- 挖坑填坑账本：主线坑、支线坑、每章期待与兑现

`02-写作计划.json` 启用 `webNovelDesign` 后，每章必须维护：

- `immersionAnchor`：本章代入锚点或现实共鸣
- `rationalizationNote`：本章如何让离奇设定看似合理
- `coreLoopStep`：本章处在核心循环哪一步
- `systemRuleUse`：本章如何使用系统、金手指、职业规则或能力规则
- `webNovelStatus`：pass / fail
- `webNovelIssues`：W-XX 问题列表

## QA 问题编号

网文顶层设计问题使用 `W-XX` 编号：

| 编号 | 类型 | 阻塞 |
|---|---|---|
| W-01 | 不合理设定缺少规则、证据或代价 | 是 |
| W-02 | 主角行动脱离熟悉领域或能力来源 | 是 |
| W-03 | 本章与核心循环脱节，像随机事件 | 是 |
| W-04 | 欲望升级不可见，读者不知道主角获得了什么 | 是 |
| W-05 | 只挖坑不填坑，或只收束不制造下一步期待 | 是 |
| W-06 | 系统/金手指/能力规则随意，像作者临时开挂 | 是 |
| W-07 | 前三章没有快速抛出核心梗、优势来源和初始冲突 | 是 |
| W-08 | 对手低智或世界无反应，导致爽感廉价 | 否 |

W 类问题修复时，优先补规则、代价、证据、人物动机和行动过程；不要只在旁白里解释“这是合理的”。
