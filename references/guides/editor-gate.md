# Editor Gate：编辑审稿门禁

Editor Gate 是章节 QA 之后的商业可读性审稿。它不替代文学质量、追读力、爽文专项和网文顶层设计，而是用编辑视角回答一个问题：读者读到这里，还想不想继续？

## 角色边界

Editor Gate 只做判断和修复指令，不直接改正文。

它重点看：

- 开头是否抓人。
- 章节主冲突是否及时出现。
- 中段是否空转。
- 主角是否有魅力和行动。
- 主角是否先有人的反应，再用能力应对。
- 配角是否制造现实压迫，而不是只提供流程信息。
- 本章是否有明确兑现。
- 结尾是否让人愿意继续。
- 是否存在真实读者会弃书的段落。

它不重点看：

- 句子是否“优美”。
- 设定是否解释得完整。
- 作者想表达的主题是否宏大。

## 必查读者流失点

| 检查点 | 通过标准 |
|---|---|
| 前300字 | 有行动、异常、冲突、强信息差或强情绪入口 |
| 约1000字 | 主冲突已经出现，读者知道主角现在要面对什么 |
| 中段 | 每 600-1000 字至少有一次信息、关系、资源、危险或选择变化 |
| 后半段 | 由电话、赶路、被试探、差点露馅、被低估、临时加测、签字、短答等外部事件继续推进 |
| 真实细节 | 物件、流程、地点或职业资料必须触发压力、选择、误判、验证、费用或暴露风险 |
| 爽点 | 兑现来自过程、阻力和反应，不靠旁白宣布 |
| 主角 | 主角有错愕、迟疑、愧疚、下意识撒谎、后悔、身体反应等人的反应，再有选择、能力、锋芒、代价或底色 |
| 配角 | 配角会催促、打断、质疑、提醒、拿走物件、要求签字、要求短答或造成误判 |
| 结尾 | 已经兑现一部分，并留下具体下一步期待 |

## 审稿输出

Editor Gate 必须写入 QA 报告或章节状态：

```text
editorGateStatus: pass / fail
editorGateScore: [0-100]
readerLossRisks:
- [读者可能弃书的位置和原因]
editorGateIssues:
- [E-XX 问题编号]
revisionLevel: none / prose / character / plot / structure
```

通过条件：

- `editorGateStatus == "pass"`
- `editorGateScore >= 85`
- `readerLossRisks` 为空
- `editorGateIssues` 为空
- `revisionLevel == "none"`

## E 问题编号

| 编号 | 类型 | 修复级别 | 阻塞 |
|---|---|---|---|
| E-01 | 前300字不抓人 | plot | 是 |
| E-02 | 1000字内主冲突未出现 | structure | 是 |
| E-03 | 中段空转，场景没有变化 | structure | 是 |
| E-04 | 主角没有主动选择或魅力展示 | character | 是 |
| E-05 | 本章没有可感知兑现 | plot | 是 |
| E-06 | 结尾只有硬钩子，没有本章价值 | plot | 是 |
| E-07 | 反派低智或世界无反应，爽感廉价 | character | 否 |
| E-08 | 文字能读但没有类型味道 | prose | 否 |
| E-09 | 配角只提供信息，不制造压迫 | character | 否 |
| E-10 | 首章核心梗后缺少外部风险 | plot | 是 |

## 修复分层

修复必须按问题级别处理，不能一律“润色”。

| revisionLevel | 何时使用 | 修复方式 |
|---|---|---|
| `structure` | 场景顺序、章节目标、主冲突出现时机错误 | 回到章节契约或场景卡重做，再重写正文 |
| `plot` | 情节有但缺兑现、缺阻力、缺期待 | 重写关键场景 |
| `character` | 主角动机弱、选择弱、对手降智 | 补动机、选择、代价和对手合理阻力 |
| `prose` | 结构成立但文字有 AI 味、节奏平 | humanize 和句段润色 |
| `none` | 无需修复 | 可进入 pre-mark-pass |

Editor Gate 失败后，必须按 `revisionLevel` 回到对应层级修复；结构级失败不得只做文字润色。
